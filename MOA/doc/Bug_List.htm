Known Problems in Code

July 14, 2000.

General problems with package.

1. Error Handling is not designed.

2. No classes for the different categories of data.

3. No handling of textual information included with classes.

4. No sophisticated way of saving and inter-relating images, plots and so on.

5. Limited to only one form of data analysis.

6. Not readily comparable to other programs on the market.

7. Lack of documentation as to how SAX data is gathered in group.

8. Lack of design plan or clear goals and objectives for software.

9. General lack of documentation of software, review of software, polling of user community of software and so on.

10. Doesn’t currently generate image distortion data.

 

Functions at c-level - Dezinger is slow

Dezinger will not work for large image sets due to model estimate algorithm in principle

Dezinger only has no error status report.

Dezinger only has one mode. Should support median mode, fourier mode, etc.

Image distortion correction sets boundaries of image to zero rather that NaN

Image distortion is much slower than it needs to be.

Image distortion is lossy.

Image distortion has no shortcut for integration routines.

Image distortion files do not have a reader in matlab.

Intensity correction is not into units of X-rays per pixel.

Integration routine does not support NaN.

Integration routine does not support errors.

Integration routine does not support negative radii.

Integration routine should have 2 modes and return bins.

Integration routine does not support dot product mode.

Integration routine only supports one mode of smoothing.

Tiff reading is not platform independent.

Tiff reading does not support full TV6 tag set.

No universal make or testing functions.

Documentation not complete.

At Matlab Level -

average_tag.m assumes image intensity monitor is in element 5 of the ATOD nonstandard tags as written by TV6.

average_tag.m does not read dates and times

caycegrate.m takes a very long time to decide on bowtie direction.

caycegrate.m doesn’t handle points on the +- boundary well.

circle.m is non-intuitive. Also, it only works for a 3 button mouse.

correct.m - relies on global variables.

dezing.m – only takes input as a 3D array. Should support file loading and other operations.

get_tag - only returns ATOD tag and doesn’t give any description of it.

guinier – no capacity for linear fit.

integrater.m - relies on global variables.

- doesn’t do negative radii.

- no adjustment on bin size and range.

mask.m - very unintuitive interface.

- doesn’t support editting of older masks.

- doesn’t show the final result.

- doesn’t permit zooming.

- doesn’t permit changing of display during masking.

new_calibration.m - Should check with user before deleting old calibration, and query for new parameters.

pixel_to_q.m - Doesn’t fix intensity to be right.

plotter.m - No decent help file.

- Should be subsumed into show

- doesn’t give hold, or handle access.

roi.m - A little non-intuitive.

- Doesn’t mark roi as it integrates.

- Doesn’t give median, number of pixels, mean location or so on.

slurp.m - Too monolithic.

- Doesn’t check to see if tags exist.

- Doesn’t correct to X-ray’s per pixel.

startup.m - not documented well enough.

- global variables.

show.m - not interactive once called up.

- doesn’t have log option built in.

- doesn’t support default range or image.

spec_to_phos.m - very hacky way of calibrating.
